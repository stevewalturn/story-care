# Story Pages System - Patient Engagement & Outcome Measurement

## Overview

**Story Pages** are personalized, interactive digital experiences created by therapists for their patients. They combine therapeutic narrative, visual media (images/videos), patient quotes, reflection questions, and surveys into a cohesive patient-facing journey. Story Pages are the primary mechanism for:

1. **Patient Engagement**: Sharing therapeutic insights outside of session time
2. **Qualitative Feedback**: Collecting patient reflections via open-ended questions
3. **Quantitative Data**: Measuring outcomes through structured surveys
4. **Narrative Therapy**: Helping patients visualize and reframe their stories

---

## Core Concept

> **"We live our lives through stories. These narratives shape identity, behavior, and possibility."**

Story Pages bring therapeutic sessions to life through:
- **AI-generated narratives** based on session transcripts
- **Visual media** (images, videos, scenes) representing therapeutic metaphors
- **Patient's own words** (quotes from sessions) highlighted for reflection
- **Guided reflection** via module-based questions
- **Outcome measurement** via standardized surveys

## Story Page Components

Every Story Page consists of **content blocks** arranged in sequence:

### 1. Text Blocks
- **Purpose**: Narrative content, introductions, therapeutic insights
- **Source**:
  - Manual: Therapist writes custom text
  - Auto-generated: AI creates narrative from session analysis
  - Library: Selected from patient's notes library
- **Format**: Markdown-supported text
- **Example**: "This session revealed patterns of resilience in how you navigated conflict..."

### 2. Image Blocks
- **Purpose**: Visual metaphors and therapeutic imagery
- **Source**:
  - Manual: Therapist uploads image
  - Library: Selected from patient's media library
  - Auto-generated: AI-generated images from session themes (DALL-E)
- **Settings**: Alt text, caption, size
- **Example**: Image of a person standing strong against a storm (resilience metaphor)

### 3. Video Blocks
- **Purpose**: Dynamic therapeutic scenes
- **Source**:
  - Manual: Therapist uploads video
  - Library: Selected from patient's media library
  - Scene: Assembled from scene editor (clips + narration + music)
- **Settings**: Autoplay, controls, captions
- **Example**: 60-second video showing therapeutic journey visualization

### 4. Quote Blocks
- **Purpose**: Highlight patient's own words from sessions
- **Source**:
  - Auto-extracted: AI extracts meaningful quotes from transcripts
  - Manual: Therapist selects from quote library
- **Format**: Italicized blockquote with attribution
- **Example**: *"I finally felt heard for the first time in years."* — From session on 2024-10-15

### 5. Reflection Blocks (Qualitative Data Collection)
- **Purpose**: Patient self-reflection and narrative exploration
- **Source**: Cloned from module's reflection template
- **Question Types**:
  - `open_text`: Free-form textarea
  - `multiple_choice`: Select from options
  - `scale`: Numeric rating
  - `emotion`: Select emotion from grid
- **Patient Response**: Stored in `reflection_responses` table
- **Example Questions**:
  - "What in this story shows the part of you that refuses to quit?"
  - "How does this change the way you see your journey?"

### 6. Survey Blocks (Quantitative Data Collection)
- **Purpose**: Structured outcome measurement
- **Source**: Cloned from module's survey template
- **Question Types**:
  - `scale`: Slider (1-10) with min/max labels
  - `multiple_choice`: Radio buttons
  - `yes_no`: Binary choice
  - `emotion`: Emotion selection (visual grid)
  - `open_text`: Text feedback
- **Patient Response**: Stored in `survey_responses` table
- **Example Questions**:
  - "How much did this story resonate with you?" (1-10 scale)
  - "What emotion did you feel most strongly?" (Emotion picker)
  - "Would you like to explore this further?" (Yes/No)

## Database Schema

### story_pages
```typescript
{
  id: uuid (PK),
  patientId: uuid (FK) → users,         // Patient this page is for
  createdByTherapistId: uuid (FK) → users, // Therapist who created it
  title: varchar(255),                   // "Your Journey of Resilience"
  description: text,                     // Summary/subtitle
  status: enum,                          // 'draft', 'published', 'archived'
  moduleId: uuid (FK) → treatment_modules, // Module used for generation (nullable)
  autoGenerated: boolean,                // True if auto-generated from module
  generationSource: enum,                // 'manual', 'module_auto', 'ai_suggested'
  emailNotificationId: uuid (FK),        // Email sent to patient (nullable)
  publishedAt: timestamp,                // When published
  archivedAt: timestamp,                 // When archived
  createdAt: timestamp,
  updatedAt: timestamp
}
```

### page_blocks
```typescript
{
  id: uuid (PK),
  pageId: uuid (FK) → story_pages,
  blockType: enum,                       // 'text', 'image', 'video', 'quote', 'reflection', 'survey'
  sequenceNumber: integer,               // Display order (0, 1, 2, ...)
  mediaId: uuid (FK) → media_library,    // For image/video blocks
  sceneId: uuid (FK) → scenes,           // For video blocks (assembled scenes)
  textContent: text,                     // For text/quote blocks
  settings: jsonb,                       // Block-specific configuration
  createdAt: timestamp
}
```

### reflection_questions (linked to reflection blocks)
```typescript
{
  id: uuid (PK),
  blockId: uuid (FK) → page_blocks,      // Which reflection block this belongs to
  questionText: text,                    // "What in this story shows your strength?"
  questionType: enum,                    // 'open_text', 'multiple_choice', 'scale', 'emotion'
  options: jsonb,                        // For multiple_choice (array of strings)
  required: boolean,                     // Is this question required?
  sequenceNumber: integer,               // Question order within block
  createdAt: timestamp
}
```

### survey_questions (linked to survey blocks)
```typescript
{
  id: uuid (PK),
  blockId: uuid (FK) → page_blocks,      // Which survey block this belongs to
  questionText: text,                    // "How much did this resonate?"
  questionType: enum,                    // 'open_text', 'multiple_choice', 'scale', 'yes_no', 'emotion'
  scaleMin: integer,                     // For scale questions (e.g., 1)
  scaleMax: integer,                     // For scale questions (e.g., 10)
  scaleMinLabel: varchar(100),           // "Not at all"
  scaleMaxLabel: varchar(100),           // "Deeply resonated"
  options: jsonb,                        // For multiple_choice/emotion (array)
  required: boolean,
  sequenceNumber: integer,
  createdAt: timestamp
}
```

### reflection_responses (patient answers)
```typescript
{
  id: uuid (PK),
  patientId: uuid (FK) → users,
  pageId: uuid (FK) → story_pages,
  questionId: uuid (FK) → reflection_questions,
  responseText: text,                    // Patient's answer
  createdAt: timestamp,
  updatedAt: timestamp                   // Allows editing responses
}
```

### survey_responses (patient answers)
```typescript
{
  id: uuid (PK),
  patientId: uuid (FK) → users,
  pageId: uuid (FK) → story_pages,
  questionId: uuid (FK) → survey_questions,
  responseValue: text,                   // Text representation (e.g., "7", "Yes", "Hope")
  responseNumeric: integer,              // Numeric value for scale questions
  createdAt: timestamp                   // No updatedAt - track submission time only
}
```

## Story Page Creation Methods

### Method 1: Manual Creation
**When to Use**: Custom, one-off story pages without module structure

**Steps**:
1. Therapist navigates to `/pages` → "Create New Page"
2. Selects patient from dropdown
3. Enters title and description
4. Adds blocks one by one:
   - Click "Add Text Block" → types content or selects from library
   - Click "Add Image Block" → uploads or selects from patient's media library
   - Click "Add Video Block" → uploads or selects scene
   - Click "Add Quote Block" → selects from patient's quote library
   - Click "Add Reflection Question" → creates custom question or selects template
   - Click "Add Survey Question" → creates custom question or selects template
5. Reorders blocks via drag-and-drop
6. Saves as draft
7. Previews in patient view mode
8. Publishes → patient receives email notification

**Result**: Fully custom story page with manual content curation

---

### Method 2: Module-Based Auto-Generation
**When to Use**: Session has been analyzed with an assigned module

**Steps**:
1. **Prerequisites**:
   - Session uploaded and transcribed
   - Module assigned to session
   - AI analysis completed using module's AI prompts

2. **Generation Process**:
   - Therapist navigates to `/pages` → "Create New Page"
   - Selects patient → sees "Generate from Module" button
   - Clicks button → `ModulePageGenerator` modal opens
   - Selects session (with assigned module)
   - Modal shows:
     - Module name and domain
     - What will be generated (content, reflections, surveys, media)
     - Customization options:
       - Custom title (optional)
       - Include media from session (checkbox)
       - Send notification on publish (checkbox)
   - Clicks "Generate" → API call to `/api/modules/{id}/generate-story-page`

3. **What Gets Auto-Generated**:
   - **Title**: Custom or auto-generated from module name
   - **Text Blocks**:
     - Intro: From `aiAnalysisResult.thematicSummary`
     - Closing: From `aiAnalysisResult.clinicalInsights`
   - **Quote Blocks**: From `aiAnalysisResult.patientQuotes[]`
   - **Image Blocks**: If `includeMedia=true`, recent images from session
   - **Video Blocks**: If `includeMedia=true`, recent videos/scenes from session
   - **Reflection Block**: Questions cloned from module's `reflectionTemplateId`
   - **Survey Block**: Questions cloned from module's `surveyTemplateId`

4. **Post-Generation**:
   - Page opens in editor (status: `draft`)
   - Therapist can:
     - Edit any block (text, media, questions)
     - Add/remove blocks
     - Reorder blocks
     - Customize cloned questions
   - Preview patient view
   - Publish → patient receives email

**Result**: AI-generated story page pre-populated with session insights, module questions, and therapeutic media

### Submitting Responses

**Reflection Submission**:
- Patient fills out textareas for each reflection question
- Clicks "Submit Reflections"
- POST `/api/responses/reflection`:
  ```json
  {
    "pageId": "uuid",
    "responses": [
      { "questionId": "uuid", "responseText": "Patient's answer..." },
      { "questionId": "uuid", "responseText": "Another reflection..." }
    ]
  }
  ```
- Success message: "Your reflections have been saved"
- Therapist can view responses in dashboard

**Survey Submission**:
- Patient interacts with survey inputs:
  - Moves sliders for scale questions
  - Clicks emotion buttons
  - Selects radio buttons for yes/no
- Clicks "Submit Survey"
- POST `/api/responses/survey`:
  ```json
  {
    "pageId": "uuid",
    "responses": [
      { "questionId": "uuid", "responseValue": "7", "responseNumeric": 7 },
      { "questionId": "uuid", "responseValue": "Anger", "responseNumeric": null },
      { "questionId": "uuid", "responseValue": "Yes", "responseNumeric": null }
    ]
  }
  ```
- Success message: "Your survey has been submitted"
- Therapist can view quantitative data in dashboard

## Access Control & Security

### Role-Based Access

**Therapist**:
- ✅ Create story pages for their own patients
- ✅ Edit draft story pages
- ✅ Publish/archive story pages
- ✅ View all responses from their patients
- ❌ View other therapists' patients' story pages
- ❌ Access patients not assigned to them

**Patient**:
- ✅ View own published story pages
- ✅ Submit reflection responses
- ✅ Submit survey responses
- ✅ Edit reflection responses (before final submission deadline)
- ❌ View other patients' story pages
- ❌ Access draft story pages
- ❌ View responses from other patients

**Organization Admin**:
- ✅ View aggregate analytics across all org therapists
- ✅ View anonymized patient responses (for research/QA)
- ✅ Access all published story pages in organization
- ❌ Edit therapist-created story pages
- ❌ View identifiable patient data without consent

### HIPAA Compliance

**Audit Logging**:
- All page views logged (who accessed which page, when)
- All response submissions logged
- All therapist edits logged
- All publish/archive actions logged
- Logs stored in `patient_page_interactions` table

**Data Encryption**:
- Story page content encrypted at rest
- Patient responses encrypted at rest
- TLS for all data in transit

**Access Verification** (in API routes):
```typescript
// Example from /api/responses/reflection/route.ts
const page = await db.query.storyPagesSchema.findFirst({
  where: eq(storyPagesSchema.id, pageId)
});

// Verify patient owns this page

  return NextResponse.json({ error: 'Unauthorized' }, { status: 403 });
}

// Verify page is published (no responding to drafts)

  return NextResponse.json({ error: 'Page not published' }, { status: 400 });
}

// Log PHI access for HIPAA compliance

  userId: session.user.id,
  resourceType: 'story_page',

  action: 'submit_reflection'
```

## Best Practices

### For Therapists

1. **Start with Modules**: Use module-based generation for consistency
2. **Customize After Generation**: Edit auto-generated content to match patient's language
3. **Quality Over Quantity**: Fewer, high-quality story pages better than many generic ones
4. **Timing Matters**: Publish story pages 1-2 days after session (while fresh)
5. **Follow Up**: Reference story page responses in next session
6. **Iterate**: Update questions based on patient feedback
7. **Visual Matters**: Always include at least one image or video
8. **Patient Voice**: Use patient's own quotes prominently

### For Creating Effective Reflection Questions

- ✅ Open-ended: "What does this show you about..."
- ✅ Strength-based: "What resilience do you notice..."
- ✅ Future-oriented: "How might this change..."
- ❌ Avoid leading questions: "Don't you think you're strong?"
- ❌ Avoid yes/no: "Did this help you?" (use surveys instead)

### For Creating Effective Surveys

- ✅ Use scales for intensity (1-10 resonance)
- ✅ Use emotion pickers for feeling identification
- ✅ Use yes/no for binary decisions (explore further?)
- ✅ Keep surveys short (3-5 questions max)
- ❌ Don't mix qualitative and quantitative in same block
- ❌ Don't make every question required (causes abandonment)

## API Reference

### Story Pages
- `GET /api/pages` - List pages (filter by patient, therapist, status)
- `POST /api/pages` - Create page with blocks
- `GET /api/pages/{id}` - Get page details (includes blocks, questions)
- `PATCH /api/pages/{id}` - Update page (edit, reorder blocks)
- `POST /api/pages/{id}/publish` - Publish page → send email
- `POST /api/pages/{id}/archive` - Archive page

### Module-Based Generation
- `POST /api/modules/{id}/generate-story-page` - Auto-generate from module
  - Body: `{ sessionId, patientId, customTitle?, includeMedia?, sendNotification? }`
  - Returns: `{ pageId, blocksCreated, questionsCloned }`

### Questions
- `GET /api/questions/reflection?blockIds={ids}` - Get reflection questions for blocks
- `GET /api/questions/survey?blockIds={ids}` - Get survey questions for blocks

### Responses
- `POST /api/responses/reflection` - Submit reflection responses
  - Body: `{ pageId, responses: [{ questionId, responseText }] }`
- `POST /api/responses/survey` - Submit survey responses
  - Body: `{ pageId, responses: [{ questionId, responseValue, responseNumeric? }] }`
- `GET /api/responses/reflection?pageId={id}` - Get patient's reflection responses
- `GET /api/responses/survey?pageId={id}` - Get patient's survey responses

### Analytics
- `GET /api/analytics/story-pages` - Aggregate story page analytics
  - Query params: `therapistId`, `patientId`, `moduleId`, `startDate`, `endDate`
  - Returns: Engagement metrics, response rates, average scores
