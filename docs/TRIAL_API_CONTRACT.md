# Trial Patient Creation API Contract

## Overview

External clinical trial admin systems can create patients in StoryCare via this API. All trial patients are created in the **Sage Health** organization and assigned to a default placeholder therapist.

## Base URL

| Environment | URL |
|---|---|
| Local | `http://localhost:3000` |
| Production | `https://app.storycare.health` |

---

## Authentication

All requests must include an `X-API-Key` header with the trial API key.

```
X-API-Key: <TRIAL_API_KEY>
```

The key is generated by running `npm run db:seed-sage-health` and is stored in `.env.local` as `TRIAL_API_KEY`.

---

## Endpoints

### `POST /api/trial/patients`

Create a new trial patient in the Sage Health organization.

#### Request

**Headers:**

| Header | Required | Description |
|---|---|---|
| `Content-Type` | Yes | `application/json` |
| `X-API-Key` | Yes | Trial API key |

**Body:**

```json
{
  "email": "patient@example.com",
  "firstName": "Jane",
  "lastName": "Doe"
}
```

| Field | Type | Required | Constraints |
|---|---|---|---|
| `email` | string | Yes | Valid email, max 255 chars |
| `firstName` | string | Yes | 1-127 chars |
| `lastName` | string | Yes | 1-127 chars |

#### Responses

**201 Created** - Patient successfully created

```json
{
  "storycareId": "a2bc52d9-e253-4d3a-a715-3904214fa87b",
  "message": "Patient created successfully"
}
```

**200 OK** - Patient already exists (idempotent, returns existing ID)

```json
{
  "storycareId": "a2bc52d9-e253-4d3a-a715-3904214fa87b",
  "message": "Patient already exists"
}
```

**400 Bad Request** - Invalid request body

```json
{
  "error": "Invalid request body",
  "details": {
    "email": ["Invalid email address"],
    "firstName": ["Invalid input: expected string, received undefined"]
  }
}
```

**401 Unauthorized** - Missing or invalid API key

```json
{
  "error": "Unauthorized"
}
```

**500 Internal Server Error** - Server-side issue (e.g. Sage Health org not seeded)

```json
{
  "error": "Trial organization not configured"
}
```

## Example Requests

```bash
# Create patient
curl -X POST https://app.storycare.health/api/trial/patients \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <TRIAL_API_KEY>" \
  -d '{"email":"jane.doe@example.com","firstName":"Jane","lastName":"Doe"}'

# Duplicate call returns 200 with existing ID
curl -X POST https://app.storycare.health/api/trial/patients \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <TRIAL_API_KEY>" \
  -d '{"email":"jane.doe@example.com","firstName":"Jane","lastName":"Doe"}'
```

## Seeded Accounts

The seed script (`npm run db:seed-sage-health`) creates the following accounts with Firebase Auth:

| Role | Email | Password |
|---|---|---|
| Org Admin | `admin@sagehealth.com` | `SageAdmin2026!` |
| Default Therapist | `default-therapist@sagehealth.com` | `SageTherapist2026!` |

Both accounts can sign into StoryCare at `/sign-in`.

## Setup

```bash
# 1. Run the seed script (creates org, accounts, and generates API key)
npm run db:seed-sage-health

# 2. The TRIAL_API_KEY is printed to console and appended to .env.local
# 3. Share the key with the external trial admin system
```
